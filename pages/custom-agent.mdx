# 自定义智能体教程

在本教程中，我们将一步步指导您如何在 Mo AI Studio 中创建一个自定义智能体。我们将创建一个名为"技术支持专家"的智能体，它能够回答有关 Mo AI Studio 的技术问题。同时，我们将通过一个 JSON 示例来详细解释智能体的各个组成部分。事实上在 Mo AI Studio 中，每个智能体就是一个 JSON 文件

## 智能体结构概览

首先，让我们看一个智能体配置的 JSON 示例，这个示例基于 `defaultRoleCards` 中的结构：

```json
{
  "id": "tech-support-expert",
  "name": "技术支持专家",
  "constraint": "\n你是 Mo AI Studio 的技术支持专家。你的任务是回答用户关于 Mo AI Studio 平台的技术问题。\n最新版本是{{latestVersion}}, 文档内容是{{docsURL}}",
  "instruction": "根据用户的问题{{input}}，提供准确、简洁的回答。如果不确定，请诚实地表示并提供寻找答案的方向。",
  "variables": {
    "latestVersion": {
      "name": "最新版本",
      "setter": "textSetter",
      "value": "2.0"
    },
    "docsURL": {
      "name": "文档URL",
      "setter": "urlSetter",
      "value": "https://docs.moaistudio.com"
    }
  },
  "outputProcessor": "defaultTextProcessor",
  "isDefault": false,
  "baseModel": "gpt-4",
  "temperature": 0.7,
  "customInstructions": [
    {
      "prefix": "@version",
      "instruction": "请提供 Mo AI Studio 的最新版本号。"
    },
    {
      "prefix": "@features",
      "instruction": "列出 Mo AI Studio 的主要功能。"
    }
  ]
}
```

现在，让我们逐步创建这个智能体，同时解释 JSON 中的每个字段。

## 步骤 1：创建新智能体

1. 登录 Mo AI Studio 平台
2. 点击"创建新智能体"按钮
3. 输入智能体名称："技术支持专家"
4. 提供简短描述："专门回答 Mo AI Studio 相关技术问题的 AI 助手"

这些信息对应 JSON 中的 `id` 和 `name` 字段。`id` 是系统内部使用的唯一标识符，而 `name` 是用户可见的智能体名称。

## 步骤 2：选择基座模型

1. 从可用的基座模型列表中选择 GPT-4
2. 点击"下一步"

这对应 JSON 中的 `baseModel` 字段。基座模型决定了智能体的基础能力。

## 步骤 3：定义指令

在指令文本框中输入以下内容：

```
你是 Mo AI Studio 的技术支持专家。你的任务是回答用户关于 Mo AI Studio 平台的技术问题。请遵循以下指南：

1. 保持专业和友好的态度
2. 提供准确和最新的信息
3. 如果不确定答案，诚实地表示并提供寻找答案的方向
4. 使用简洁明了的语言，避免技术行话
5. 在适当的时候提供相关文档的链接
```

这对应 JSON 中的 `constraint` 字段。`constraint` 定义了智能体的行为准则和基本设定。

## 步骤 4：设置约束

在约束文本框中输入：

```
1. 不讨论 Mo AI Studio 的内部开发计划或未公开的功能
2. 不分享用户的个人信息或账户详情
3. 不对竞争产品发表评论
4. 在回答中保持中立和客观
```

这些约束也是 `constraint` 字段的一部分，它们进一步限定了智能体的行为边界。

## 步骤 5：添加变量

1. 点击"添加变量"按钮
2. 创建一个名为"最新版本"的文本变量，初始值设为"2.0"
3. 创建一个名为"文档 URL"的 URL 变量，值设为"https://docs.moaistudio.com"

这对应 JSON 中的 `variables` 字段。变量允许你存储和更新智能体可能需要的动态信息。

## 步骤 6：配置输出处理器

1. 选择"默认文本处理器"
2. 设置最大输出长度为 500 个词

这对应 JSON 中的 `outputProcessor` 字段。输出处理器决定了如何处理和格式化智能体的回答。

## 步骤 7：调整温度设置

将温度值设置为 0.7，以平衡一致性和创造性

这对应 JSON 中的 `temperature` 字段。温度值控制着输出的随机性和创造性。

## 步骤 8：添加自定义指令

1. 点击"添加自定义指令"按钮
2. 添加以下自定义指令：
   a. 前缀：@version
   指令内容：请提供 Mo AI Studio 的最新版本号。
   b. 前缀：@features
   指令内容：列出 Mo AI Studio 的主要功能。

这对应 JSON 中的 `customInstructions` 数组。自定义指令允许用户触发特定的行为或获取特定的信息。

## 步骤 9：保存和测试

1. 点击"保存智能体"按钮
2. 在测试界面中，输入一些示例问题：
   - "如何在 Mo AI Studio 中创建新智能体？"
   - "Mo AI Studio 支持哪些基座模型？"
   - "如何设置智能体的约束？"
   - "@version"
   - "@features"

## 步骤 10：优化和迭代

基于测试结果，可能需要：

1. 调整指令以覆盖更多场景
2. 增加或修改约束
3. 添加更多相关变量
4. 微调温度设置
5. 修改或添加自定义指令

## 步骤 11：导出和分享

1. 点击"导出智能体"按钮
2. 选择导出格式（JSON）
3. 保存导出的文件
4. 可以通过导入功能在其他 Mo AI Studio 实例中使用该智能体

## 结论

通过这个教程，您已经成功创建了一个自定义的"技术支持专家"智能体。这个过程展示了如何利用 Mo AI Studio 的各种功能来创建一个专门的 AI 助手。

JSON 配置文件中的每个字段都有其特定的作用：

- `id` 和 `name`：定义智能体的唯一标识符和显示名称
- `constraint`：设置智能体的行为准则和基本设定
- `instruction`：提供智能体执行任务的具体指导
- `variables`：存储可动态更新的信息
- `outputProcessor`：决定如何处理和格式化输出
- `isDefault`：标识是否为默认智能体
- `baseModel`：选择底层的语言模型
- `temperature`：控制输出的随机性
- `customInstructions`：定义特殊的触发指令

记住，创建优秀的 AI 智能体是一个迭代的过程。不断收集用户反馈，分析性能数据，并据此改进您的智能体定义。祝您在 Mo AI Studio 的探索之旅愉快且富有成效！
