Mo 不只是一个被动的回答机器，它还具备强大的指令系统，让你的开发工作事半功倍：

1. **@dev 指令**：当你输入 @dev 指令时，Mo 就会进入开发者模式。它会根据你的项目文件和上下文，给出精准的代码修改建议。想象一下，有一个资深开发者实时review你的代码，是不是很酷？

2. **自定义指令**：Mo 允许你定义自己的指令前缀。比如，你可以设置 @optimize 指令来专门优化代码，或者 @explain 指令来详细解释某个概念。这就像是给 Mo 装备了各种特殊技能，让它更贴合你的工作方式。

### Mo 的百宝箱：设置器

Mo 配备了一系列强大的设置器，让数据输入变得既灵活又精准：

1. **文本设置器**：最基本但最常用的设置器。无论是简单的配置还是复杂的指令，都能轻松输入。

4. **URL设置器**：直接从网上获取信息。比如，你可以让 Mo 阅读最新的技术文档，然后为你解释。

5. **日期设置器**：处理时间相关的任务变得轻而易举。无论是计划项目时间线还是处理日期相关的代码，Mo 都能帮你搞定。

### Mo 在行动：实际使用场景

1. **代码诊断医生**：
   遇到棘手的 bug？把相关代码通过文件设置器发送给 Mo，然后说："@dev 这段代码有个性能问题，能帮我找出来并优化吗？" Mo 会仔细分析代码，找出问题所在，并给出优化建议。

2. **项目架构师**：
   开始新项目时，使用目录设置器上传你的项目结构，然后问 Mo："基于这个结构，你觉得我们的前端架构还有哪些可以改进的地方？" Mo 会根据最佳实践给出建议，就像有一个架构师在为你把关。

3. **API 集成专家**：
   需要集成新的 API？使用 URL 设置器发送 API 文档给 Mo，然后说："帮我写一个 React Hook 来调用这个 API。" Mo 会为你生成代码，并解释每一步的作用。

4. **CSS 魔法师**：
   遇到难以实现的布局？描述你想要的效果，然后说："@dev 用 Tailwind 实现这个布局。" Mo 会为你编写精确的 Tailwind 类名，并解释每个类的作用。

5. **敏捷教练**：
   使用日期设置器设置项目时间线，然后让 Mo 帮你规划 sprint："基于这个时间线，帮我制定一个两周的 sprint 计划。" Mo 会考虑节假日、团队能力等因素，给出合理的计划建议。
